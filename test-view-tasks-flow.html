Problem Summary

Issue: When clicking "View Tasks" on member cards in members.php, the admin dashboard shows all tasks instead of filtering by the specific team member.

Root Cause: The admin-dashboard.php was not reading the user_id parameter from the URL and was calling getTasks(null, $selectedDate) which returns all tasks.

‚úÖ Fix Applied

Changes Made:

  • Added user_id parameter parsing in admin-dashboard.php
  • Updated getTasks call to use filtered user ID
  • Updated page titles to show filtered user name
  • Added proper validation for user_id parameter
